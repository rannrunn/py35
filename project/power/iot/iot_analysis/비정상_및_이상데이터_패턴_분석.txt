○ 데이터 분류
    1) 정상 데이터 : 정상적으로 측정된 데이터
        1> 이상하지 않은 데이터 :
        2> 이상 데이터 :
    2) 비정상 데이터 : 센서가 일시적으로 오작동하여 측정된 데이터
    3) 노이즈 데이터 : 데이터 측정 시 다른 요인에 의한 간섭을 받은 것
    4) 판단불가 데이터
        1> 이상 데이터와 비정상 데이터에 대한 판단이 불가한 데이터
        2> 이상하지 않은 데이터와 노이즈 데이터에 대한 판단이 불가한 데이터

○ 비정상데이터 처리를 위한 작업 진행 과정
    1. 비정상데이터를 제거하기 위해 표본의 평균, 표준편차, 정규분포, 차분 및 이동평균을 사용하기로 결정
        ＠ 평균, 표준편차, 정규분포, 차분, 이동평균에 대한 이미지 표시

    2. 정확한 표본 통계량을 구하기 위해서는 비정상데이터가 제거된 데이터가 필요
        ＠ 표본 통계량 관련 이미지 표시

    3. 비정상데이터가 제거된 정확한 표본 통계량을 구하기 위해서는 히스토그램이나 클러스터링 등을 이용해 제거하는 방법을 생각해 볼 수 있음

    4. 클러스터링을 사용할 경우 파라미터 조정작업에 시간이 들 수 있고, 클러스터링 결과에 대한 의미를 도출하기 위해선
       결국 사람이 데이터를 확인하는 작업이 필요하기 때문에 히스토그램을 기반으로 비정상 데이터를 분류
        ＠ 클러스터링 진행 프로세스 그림으로 표시

    5. 데이터를 이미지화 하여 비정상데이터를 분석하고 분류하는 작업을 진행
        ＠ 이미지화 한 비정상데이터 예 표시

    6. 아래와 같은 케이스의 비정상데이터를 발견
        1) 여러 변수에 특정 수치가 동시에 측정되는 세 가지 경우 (광주에 설치된 알프스의 1차 5개 센서에서 약 85,000 개 데이터에서 발생)
            ＠ 오류 데이터 대한 테이블 표시
        2) 스펙 범위 밖 데이터가 측정되는 경우
            ＠ 측정 범위에 대한 테이블 표시
            ＠ 비정상데이터 이미지 예 표시
        3) 값이 이전 시점에 비해 큰폭으로 증가하였다가 원래 값으로 돌아오는 경우나 그 반대 상황이 발생하는 경우
            ＠ 이미지 예 표시 (온도부터 UVC까지 모두 표시하면 좋을 듯)
        4) 특정 수치가 측정된 후 원래 수치로 돌아오는 경우
            1> 온도에 0 도가 측정되는 경우
                ＠ 비정상데이터 이미지 예 표시
            2> GEOMAG_X, GEOMAG_Y, GEOMAG_Z 에 0 uT 가 측정되는 경우
                ＠ 비정상데이터 이미지 예 표시
            3> VAR_X, VAR_Y, VAR_Z 에 0 mg 가 측정되는 경우
            ※ 위 세가지 경우는 프로그램에서 변수를 먼저 0 으로 초기화한 다음에 센서 측정치를 업데이트 하게되는데 업데이트하지 않고 전송한 데이터로 추정됨
                ＠ 비정상데이터 이미지 예 표시
            4> 유중온도에 120 도가 측정되는 경우
                ＠ 비정상데이터 이미지 예 표시

    7. 각 케이스에 대한 비정상데이터 자동 처리 방법
        1) 여러 변수에 특정 수치가 동시에 측정되는 세 가지 경우
            -> 특정 수치가 측정된 변수들 외에 다른 변수에서 측정된 데이터는 정상 수치로 보이는 경우도 있으나 데이터가 적고 신뢰하기 힘드므로 행 전체 삭제
        2) 스펙 범위 밖 데이터가 측정되는 경우
            스펙 상의 범위 밖 데이터는 모두 np.nan 대체
            예외> 조도, UV, USN, UVC 의 경우 임계치를 통해 이벤트를 판단하기 때문에 상한을 벗어나는 데이터가 있어도 처리하지 않음
            주의> 차수별, 제작사별 센서 측정 범위에 대한 조사를 정확하게 해야함
        3) 값이 이전 시점에 비해 큰폭으로 증가하였다가 원래 값으로 돌아오는 경우나 그 반대 상황이 발생하는 경우
            온도 : 1차 차분 후 비정상데이터를 검출하고 np.nan 대체 -> 앞 뒤 값의 평균으로 보상하고 보상이 불가능한 경우 보상하지 않음
            습도 : 측정 범위 내에서 데이터의 변화폭이 크기 때문에 비정상데이터를 판단하기 어려움 -> 처리하지 않음
            피치 : 이전까지 정상적 센싱되고 있던 상태에서 발생했다면 일시적인 센서 오류이거나 사람의 영향일 수 있음 -> 차분을 이용해 처리
            롤 : 이전까지 정상적 센싱되고 있던 상태에서 발생했다면 일시적인 센서 오류이거나 사람의 영향일 수 있음 -> 차분을 이용해 처리
            조도 : UV 발생 여부를 파악하여 비정상데이터 처리
            자외선 : 조도 발생 여부를 파악하여 비정상데이터 처리
            대기압 : 1차 차분 후 비정상데이터를 검출하고 np.nan 대체 (고도에 따라 변할 수 있으므로 피치, 롤의 변화가 없는 상태에서 대기압이 크게 변화했다면 비정상데이터일 확률이 높음)
                     -> 앞 뒤 값의 평균으로 보상하고 보상이 불가능한 경우 보상하지 않음
            베터리 : 1차 차분 후 비정상데이터를 검출하고 np.nan 대체 -> 앞 뒤 값의 평균으로 보상하고 보상이 불가능한 경우 보상하지 않음
            자기장 X : 짧은 시간 동안 값이 크게 변하는 것이 가능하므로 센서 오류인 지 실제 측정값인 지 IoT 데이터만으로 비정상데이터를 판단하기 힘듦 -> 처리하지 않음
            자기장 Y : 자기장 X와 동일
            자기장 Z : 자기장 X와 동일
            진동 X : 자기장 X와 동일
            진동 Y : 자기장 X와 동일
            진동 Z : 자기장 X와 동일
            초음파 : 부분방전이 발생했을 때 초음파와 자외선-C가 항상 동시에 나타난다면 자외선-C 데이터를 이용하여 비정상데이터를 처리하는 방안을 생각해 볼 수 있지만
                     이 것은 자외선-C 데이터를 신뢰할 수 있다는 전제 하에서만 가능하므로 초음파 발생 데이터는 비정상데이터 탐지 및 처리를 하지 않음
            유중온도 : 1차 차분 후 비정상데이터를 검출하고 np.nan 대체 -> 앞 뒤 값의 평균으로 보상하고 보상이 불가능한 경우 보상하지 않음
            자외선-C : 초음파와 동일
        4) 특정 수치가 측정된 후 원래 수치로 돌아오는 경우
            -> 각 변수별로 처리
            1> 온도 0 : 1차 차분 값과 벤드를 이용해 비정상데이터를 판별하여 np.nan 대체 -> 앞 뒤 값의 평균으로 보상하고 보상이 불가능한 경우 보상하지 않음
            2> 자기장 0 : 1차 차분 값을 이용해 비정상데이터를 판별하여 np.nan 대체 -> 앞 뒤 값의 평균으로 보상하고 보상이 불가능한 경우 보상하지 않음
                          ※ 3) 번 항목의 자기장은 처리하지 않지만 이 경우 비정상데이터일 확률이 높기 때문에 처리
            3> 진동 0 : 자기장 0과 동일
            4> 유중온도 120 : 온도 0과 동일

    8. 비정상데이터 발생 시 사용자에게 알람을 주는 방법
        -> 고장 발생의 경우와 구분하여 사용자에 알람을 주는 방안을 마련

    9. 비정상데이터 전처리 결과에 대한 데이터베이스 처리
        -> IoT 데이터 분석을 위해서는 IoT Service 데이터베이스 외에 분석을 위한 데이터베이스를 추가로 구성해야 함
            @ 데이터베이스 구성도 추가
        -> 파싱 후 데이터를 저장하는 테이블에 자동 증가열을 추가하여 유니크 키로 사용하고 이 키를 외래키로 사용하는 테이블을 만들어 전처리 후의 값을 저장
            @ 테이블 구성도 추가
        -> 사용자에게는 전처리 후의 값이 보여지도록 함
            @ 시각화 그림 추가
